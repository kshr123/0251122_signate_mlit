# EDA実践ガイド

> 4フェーズで進めるEDA標準フロー

---

## 📊 EDA 4フェーズ

### Phase 1: データ概要把握
- 形状（行数・列数）
- カラム名・データ型
- 欠損値の有無
- 基本統計量

### Phase 2: 目的変数の分析
- ヒストグラム（分布）
- Box plot（外れ値）
- Q-Qプロット（正規性）
- Train/Test分布比較

### Phase 3: 特徴量の分析
- **数値**: 分布、相関行列、目的変数との散布図
- **カテゴリ**: 頻度分布、カテゴリ別目的変数Box plot

### Phase 4: データ品質チェック
- 欠損パターン（missingno）
- 重複レコード
- Train/Test重複
- データリーク検出（相関 > 0.99）

---

## ✅ 必須可視化チェックリスト

| 対象 | 可視化 |
|------|--------|
| 目的変数 | ヒストグラム, Box plot, Q-Q |
| 数値変数 | 分布一覧, 相関行列 |
| カテゴリ変数 | 頻度棒グラフ, 目的変数別Box plot |
| データ品質 | 欠損パターン |

---

## 📝 EDA完了後のアクション

1. **前処理方針**: 欠損補完、外れ値対処
2. **特徴量方針**: エンコーディング、交互作用
3. **モデリング方針**: ベースライン、CV戦略
4. **リスク確認**: データリーク、Train/Test差異
